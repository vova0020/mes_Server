# Скрипт получения полной информации по заказу

## Описание

Скрипт `get_order_full_info.ts` выводит в консоль полную информацию по закрытому (или любому) заказу, включая:

### Что выводится:

1. **Основная информация о заказе**
   - Номер партии, название, статус
   - Процент выполнения, приоритет
   - Даты создания, обновления, завершения

2. **История статусов заказа**
   - Все изменения статуса с датами
   - Кто изменил статус

3. **Упаковки заказа**
   - Код, название, количество
   - Статус упаковки
   - Исходный состав (детали из справочника)
   - Производственные детали с их статусами

4. **Назначения поддонов на упаковки**
   - Какие поддоны назначены
   - Количество (исходное, использованное)
   - Статус назначения

5. **Задачи упаковки**
   - Станок, статус, приоритет
   - Назначенное и выполненное количество
   - Кто выполнял, когда

6. **Поддоны по деталям**
   - Название, количество, активность
   - Размещение в буферах (история)
   - Назначения на станки
   - Прогресс по этапам маршрута
   - История перемещений (откуда-куда, кем)
   - История операций на станках (длительность, оператор)

7. **Рекламации и брак**
   - Количество брака, статус
   - Этап и станок, где обнаружен брак
   - Кто сообщил и подтвердил
   - Причины брака (типы дефектов)
   - Вложения (фото/файлы)
   - Движения/возвраты деталей
   - История обработки рекламации

8. **Движения запасов**
   - Все изменения количества деталей
   - Причины движений
   - Связь с рекламациями
   - Возвраты на этапы/станки

9. **События по заказу**
   - Все события из системы аудита
   - Тип события, сущность
   - Старые и новые значения
   - Кто выполнил действие

## Запуск

### Вариант 1: Через ts-node (рекомендуется для разработки)

```bash
npx ts-node scripts/get_order_full_info.ts
```

### Вариант 2: Компиляция и запуск

```bash
# Компиляция
npx tsc scripts/get_order_full_info.ts --outDir dist --esModuleInterop --resolveJsonModule --skipLibCheck

# Запуск
node dist/scripts/get_order_full_info.js
```

### Вариант 3: Добавить в package.json

Добавьте в секцию `scripts`:

```json
"scripts": {
  "order-info": "ts-node scripts/get_order_full_info.ts"
}
```

Затем запускайте:

```bash
npm run order-info
```

## Изменение ID заказа

По умолчанию скрипт выводит информацию по заказу с ID 73. Чтобы изменить ID:

1. Откройте файл `scripts/get_order_full_info.ts`
2. Найдите в конце файла строку:
   ```typescript
   const orderId = 73;
   ```
3. Измените число на нужный ID заказа
4. Сохраните и запустите скрипт

## Перенаправление вывода в файл

Чтобы сохранить результат в файл:

```bash
npx ts-node scripts/get_order_full_info.ts > order_73_report.txt
```

Или с кодировкой UTF-8 (для Windows):

```bash
npx ts-node scripts/get_order_full_info.ts | Out-File -Encoding UTF8 order_73_report.txt
```

## Требования

- Node.js
- Prisma Client
- Доступ к базе данных (настроенный DATABASE_URL в .env)

## Примечания

- Скрипт автоматически отключается от базы данных после выполнения
- Все даты выводятся в формате локали ru-RU
- Длительность операций выводится в минутах и секундах
- Если данных нет (например, нет рекламаций), выводится соответствующее сообщение
